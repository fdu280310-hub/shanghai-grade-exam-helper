# 上海等级考出题助手

将外地（全国卷、北京卷等）物理/化学题目一键转换为上海等级考风格的 Word 文档。

## 核心能力

1. **识别题目**：从图片（截图/拍照）或 PDF 中读取外地考试题目内容
2. **情景化改编**：为缺少情境的题目添加上海等级考标志性的"情景引入"（工业应用、科研成果、日常生活等真实场景）
3. **格式规范输出**：生成完全符合上海等级考排版规范的 .docx 文件

## 何时使用

**合适的场景**：
- 教师在网上看到外地好题（北京卷/全国卷/江苏卷等），想改编到周练/月考卷中
- 需要快速将一本外地教辅中的题目改成上海风格
- 需要为某个知识点补充等级考风格的练习题
- 出月考卷/周练卷时需要统一格式

**不合适的场景**：
- 需要生成整份完整试卷（本 skill 聚焦于单个大题或几个大题的改编）
- 小学/初中题目（上海等级考特指高中等级考）
- 语文/数学/英语/生物等其他科目（当前仅支持物理和化学）

## 工作流

### Step 1: 接收题目与素材研读

从用户获取输入：
- **图片**：截图或拍照的外地题目。优先让用户提供清晰图片。
- **PDF**：包含题目的 PDF 文件。需要用户指定页码或大致位置。
- **纯文字**：用户直接复制粘贴的题目文本。

识别并确认：
- 科目（物理 or 化学）
- 题目结构和考点
- 图表描述等

**若用户未提供素材**：利用搜索引擎查找契合中学化学/物理的知识点作为素材（如最新科研成果、工业应用、环保案例、生活场景等）。

**素材研读**：仔细阅读用户提供的全部素材，提取关键信息点——核心反应/原理、数据、工艺流程、科学概念等。这些信息将作为后续出题的知识基础。

### Step 2: 知识准备

根据用户要求和素材内容，借助自身知识储备以及搜索引擎进一步充实知识准备：
- 化学 → 阅读 `references/chemistry_standards.md`，确定相对原子质量、化学式规范
- 物理 → 阅读 `references/physics_standards.md`，确定物理量符号、单位规范
- 阅读 `references/scenario_examples.md` 获取情景引入灵感
- 搜索相关的前沿应用、生活案例，丰富情景素材

### Step 3: 构建递进式题目组

**核心原则**：依据用户提供的素材和搜索结果，精心构思一组知识、能力逐步加深的化学/物理作业题。题目类型应丰富多样，涵盖但不限于填空题、单选题、双选题、简答题、综合分析题等。

#### 3.1 前10题认知层级递进（基于素材/教材知识）

| 题号 | 题型 | 认知层级 | 说明 |
|------|------|----------|------|
| 1 | 填空 | **知道** | 源自素材或教材本身的基础知识填空，直接提取信息即可作答 |
| 2 | 单选 | **理解** | 把第1题与多个先验知识联系起来，每个选项聚焦素材的一个知识点 |
| 3 | 填空 | **理解** | 把素材与宏观现象或微观本质相联系 |
| 4 | 双选 | **理解** | 把素材与变化观念或平衡思想相联系 |
| 5 | 简答 | **理解** | 把素材与模型认知或证据推理相联系 |
| 6 | 简答 | **运用** | 把素材与科学探究或创新意识相联系 |
| 7 | 单选/填空 | **理解→运用** | 素材知识在新情境中的迁移应用 |
| 8 | 双选 | **运用** | 多知识点综合判断 |
| 9 | 填空/简答 | **运用→综合** | 数据处理、图表分析或计算推理 |
| 10 | 简答/计算 | **综合** | 开放性问题或综合性计算，需要综合运用多知识点 |

#### 3.2 改编规则

- 阅读 `references/question_adaptation_guide.md` 了解改编规则
- 对每个大题目：
  - 生成1-2段楷体情景引入（真实、科学、与题目自然衔接）
  - 重新编号：大题"一、二、…"，子题"1. 2. 3. …"
  - 选项格式：A．B．C．D．，不定项标注"（不定项）"
  - 确保难度匹配等级考要求（重理解重推理，计算量适中）
- 先以 Markdown 格式展示改编后的完整题目给用户确认，再生成 docx

#### 3.3 题型要求

- **单选题**：4个选项，仅1个正确。不特别标注。
- **双选题（不定项）**：4个选项，1~2个正确。标注"（不定项）"。只有1个正确选项时多选不给分；有2个正确选项时漏选1个得一半分，错选不得分。
- **填空题**：留空行，关键位置用"__________"标记。
- **简答题**：留3-4行作答空间。侧重原因分析、方案设计、推理说明。
- **计算题**：留充分作答空间。侧重公式推导和数值计算。

### Step 4: 生成 Docx

确认后，构建 JSON 数据并调用生成脚本：

```bash
python "<SKILL_ROOT>/scripts/generate_docx.py" <data.json> <output.docx>
```

JSON 数据结构参见 `scripts/generate_docx.py` 中的 `SAMPLE_DATA`。关键字段：
- `subject`: "化学" or "物理"
- `title`: 试卷标题（如"高三化学练习"）
- `subtitle`: 副标题/系列名（如"高三教学资源包"）
- `instructions`: 考生注意条目列表
- `molar_mass`: 相对原子质量（化学专用）
- `big_questions[]`: 大题列表，每个包含：
  - `number`: "一"/"二"/"三"
  - `topic`: 大题主题
  - `scenario[]`: 情景引入段落（楷体）
  - `sub_questions[]`: 子题列表，每个包含 `number`, `stem`, `type`, `options`(选择题), `indefinite`(是否不定项)

### Step 5: 验证与交付

生成后检查：
- 副标题黑体16pt、标题黑体16pt居中
- 情景引入楷体10.5pt首行缩进
- 英文/数字是否使用 Times New Roman
- 化学式上下标是否正确（Cr(OH)₃ → subscript 3, SO₄²⁻ → superscript 2-）
- 选项是否通过 tab 分隔对齐
- 页面是否为 A4

交付文件给用户，支持根据反馈迭代修改。

## 格式规范速查

详细规范见 `references/formatting_standards.md`。

| 元素 | 字体 | 字号 | 对齐 | 行距 | 特殊 |
|------|------|------|------|------|------|
| 副标题(系列名) | 黑体 | 16pt | 居中 | 360 | 英文/数字用 Times New Roman |
| 标题(试卷名) | 黑体 | 16pt | 居中 | 360 | 英文/数字用 Times New Roman |
| 大题标题 | 黑体 | 14pt bold | 居中 | 360 | "一、铬及其化合物的工业应用" |
| 考生注意标题 | 黑体 | 10.5pt bold | 左 | 293 | "考生注意：" |
| 考生注意条目 | 黑体 | 10.5pt | 左(首行缩进420) | 293 | 编号用 TNR |
| 情景引入 | **楷体** | 10.5pt | 左(首行缩进480) | 293 | 英文/数字用 Times New Roman |
| 小题题干 | 黑体(编号)+宋体(题干) | 10.5pt | 左(悬挂缩进420) | 360 | 编号用 TNR |
| 选项 | 宋体 | 10.5pt | 左(两行A-B/C-D，tab分隔) | 360 | 编号用 TNR |
| 相对原子质量 | 黑体 | 10.5pt | 左(首行缩进420) | — | 数值用 TNR |

页面：A4 (11906×16838 twips)，页边距上/下 1440，左/右 1800。

### 化学式规范

化学式中的 **下标** 使用 OOXML `<w:vertAlign w:val="subscript"/>`，**上标** 使用 `<w:vertAlign w:val="superscript"/>`。用户在 JSON 数据中使用 Unicode 上下标字符（如 ₃²⁻）即可，生成脚本自动转换为正确的 OOXML 格式。

## 科目规范要点

### 化学

- 文档开头须包含"相对原子质量"行（读 `references/chemistry_standards.md` 获取标准列表）
- 化学式中的数字用下标 `<w:vertAlign w:val="subscript"/>`
- 离子电荷用上标 `<w:vertAlign w:val="superscript"/>`
- 可逆反应用 ⇌ (U+21CC)
- 常见情景主题：工业合成、废水处理、电池技术、有机合成路线

### 物理

- 物理量用斜体，单位用正体
- 矢量用粗体或上方加箭头
- 图表需带坐标轴标注
- 有效数字通常保留2-3位
- 常见情景主题：交通工具、航空航天、体育运动、医疗设备

## 注意事项

1. **字体兼容**：模板使用 Windows 中文系统自带的 宋体、黑体、楷体。在 Mac/Linux 上编辑时可能会降级为系统默认字体，但打印效果通常不受影响。
2. **化学公式**：简单化学式用 Unicode + 上下标 XML 解决。极端复杂的结构式建议在 Word 中手动插入 ChemDraw 对象。
3. **图表处理**：原题中的图表描述保留在文字中。如需插入图片，在生成后在 Word 中手动操作。
4. **WPS 兼容**：模板来源于 WPS 创建的文档，在 WPS 中打开效果最佳，Microsoft Word 中也完全兼容。
